LOADING 'Gif'

Internet speeds are not always ideal so most developers use a loading gif to indicate that the image is loading (most of the time).

Gifs don't really have a good frame rate especially on mobile devices.

CSS3 animations run at 60fps because the browser loads it on the gpu. If you make a loading gif using HTML the browser has one less thing to download.

Concept

1

To make the ball have a cartoony feel, need to exaggerate gravity. So the ball stretches in the y direction on the way down. It also speeds up as it reaches the ground.

2

The ball is squashes under its own weight to have a jelly/rubber feel. y decreases and x increases. Time spent will be the greatest at this point.

3

This is the mirror of animation of 1.

I start with a sketch of each frame of the gif (usually doodles on my phone), as I go along I think about how to code the css.

I then write the HTML. I nest divs inside a container div. The nested divs have the same class so I can target them with css selectors (nthchild). I set the animation iteration count to 'infinite' so it repeats. So the instead of appending a Gif image you just append the HTML.
