ANIMATED BUTTONS

Since CSS3 transitions and animations (some properties) use the gpu of a given device we can have beautiful 60fps animations and free up the cpu to do what it was intended for.

We can animate any element meaningfully into the screen giving the user the best experience possible on browsers.

I decide to build buttons was pure CSS and HTML buttons. So no downloading of sprites for button backgrounds-image or font icons.

I did this only with CSS transform properties to use the gpu.

Skip to demo

Concept

Hamburger Menu

With CSS order matters if I add a class dynamically which has CSS applied to it the CSS of that element will be 'overwritten'.

HTML
<div class="animated-button-example off"> <div class="rect"></div> <div class="rect"></div> <div class="rect"></div> </div>
HTML

<div class="animated-button-example off"> <div class="rect"></div> <div class="rect"></div> <div class="rect"></div> </div>

CSS
off

.off .rect:nth-child(1) { transform: translate3d(5px, 10px, 0); }

.off .rect:nth-child(2) { transform: translate3d(5px, 45px, 0); }

.off .rect:nth-child(3) { transform: translate3d(5px, 80px, 0); }

on

.on .rect:nth-child(1) { transform: translate3d(5px, 45px, 0) rotate(45deg); }

.on .rect:nth-child(2) { transform: translate3d(5px, 45px, 0) rotate(-45deg); }

.on .rect:nth-child(3) { transform: translate3d(5px, 45px, 0) rotate(-45deg); }

As seen when the button is closed the css for closed is applied but when clicked open 'overwrites' it.

To keep the sample code simple the browser prefixes have been left out.
